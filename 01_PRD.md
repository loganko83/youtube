# 제품 요구사항 문서 (PRD)
## AI 기반 자동 동영상 콘텐츠 생성 플랫폼 "AutoClip"

---

## 1. 제품 개요

### 1.1 제품 비전
**"누구나 클릭 몇 번으로 전문가 수준의 정보성 숏폼/롱폼 영상 채널을 운영할 수 있는 노코드 자동화 인프라"**

### 1.2 핵심 가치 제안
| 가치 | 설명 |
|------|------|
| **진입장벽 제거** | 기술적 역량 없이도 고품질 영상 콘텐츠 제작 가능 |
| **비용 효율성** | 전문 편집자 고용 대비 90% 이상 비용 절감 |
| **안전성 보장** | AI 환각 억제 및 플랫폼 정책 준수 자동 검증 |
| **확장성** | 다국어, 다채널, 다버티컬 동시 운영 지원 |

### 1.3 목표 시장
- **1차 타겟 (Beachhead)**: 시니어 건강정보 채널 운영자 및 요양시설
- **2차 타겟**: 금융/재테크, 테크/AI 리뷰, 역사/지식 콘텐츠 크리에이터
- **3차 타겟 (B2B)**: 기업 브랜드 저널리즘, 부동산 중개, 공공기관

## 3. 기능 요구사항

### 3.1 핵심 기능 (MVP)

#### F1. 콘텐츠 설정 위저드
```
우선순위: P0 (필수)
설명: 사용자가 원하는 콘텐츠 유형을 단계별로 선택

입력 항목:
├── 대상 시장 (버티컬)
│   ├── 시니어 건강
│   ├── 금융/재테크
│   ├── 테크/AI
│   ├── 역사/지식
│   └── 커머스/리뷰
├── 콘텐츠 형태
│   ├── 숏폼 (60초 이하)
│   ├── 미드폼 (1-5분)
│   └── 롱폼 (5분 이상)
├── 톤앤매너
│   ├── 전문적/신뢰감
│   ├── 친근한/캐주얼
│   ├── 긴급한/뉴스형
│   └── 미스터리/스토리텔링
├── 언어
│   ├── 한국어
│   ├── 영어
│   ├── 일본어
│   └── 다국어 동시
└── 업로드 플랫폼
    ├── YouTube Shorts
    ├── TikTok
    ├── Instagram Reels
    └── 모든 플랫폼
```

#### F2. 데이터 소스 연동
```
우선순위: P0 (필수)
설명: 콘텐츠 원본 데이터를 자동으로 수집

지원 소스:
├── 외부 API
│   ├── Yahoo Finance (금융 데이터)
│   ├── Google News (뉴스 피드)
│   ├── PubMed RSS (의학 정보)
│   └── CoinGecko (암호화폐)
├── 채널 기반
│   ├── YouTube Channel ID (하이라이트 추출)
│   └── RSS Feed URL
├── 직접 입력
│   ├── 주제 키워드
│   ├── 원본 텍스트
│   └── 참조 URL
└── 기업용
    ├── 블로그 RSS
    ├── LinkedIn Page
    └── 보도자료 링크
```

#### F3. AI 콘텐츠 생성 엔진
```
우선순위: P0 (필수)
설명: n8n + Gemini 기반 자동 대본 및 영상 생성

파이프라인:
1. 데이터 수집 (Cron/Webhook 트리거)
2. 콘텐츠 분석 및 요약 (Gemini 2.5 Flash)
3. 대본 작성 (Gemini 3 Flash)
   - Hook (처음 3초)
   - Body (핵심 내용)
   - CTA (행동 유도)
4. 안전성 검증 (Safety Filter Agent)
5. 이미지/영상 소스 선정
6. TTS 음성 생성 (ElevenLabs/OpenAI TTS)
7. 영상 렌더링 (Creatomate/Shotstack)
8. 메타데이터 생성 (제목, 설명, 태그)
```

#### F4. 템플릿 관리 시스템
```
우선순위: P0 (필수)
설명: 버티컬별 최적화된 영상 템플릿 제공

템플릿 구성요소:
├── 인트로 (0-3초)
├── 본문 레이아웃
├── 자막 스타일
├── 전환 효과
├── 아웃트로 (마지막 5초)
└── 브랜드 요소
    ├── 로고 워터마크
    ├── 컬러 팔레트
    └── 폰트 설정
```

#### F5. 대시보드 및 모니터링
```
우선순위: P0 (필수)
설명: 생성된 콘텐츠 현황 및 성과 추적

표시 정보:
├── 생성 현황
│   ├── 오늘 생성 수
│   ├── 대기 중인 콘텐츠
│   ├── 승인 필요 콘텐츠
│   └── 업로드 완료
├── 성과 지표 (연동 시)
│   ├── 조회수 추이
│   ├── 구독자 변화
│   └── 예상 수익
└── 워크플로우 상태
    ├── 실행 중
    ├── 오류 발생
    └── 스케줄 예정
```

## 4. 비기능 요구사항

### 4.1 성능 요구사항
| 항목 | 요구 수준 | 측정 방법 |
|------|----------|----------|
| 영상 생성 시간 | 숏폼 60초 이내, 롱폼 5분 이내 | 렌더링 완료까지 시간 |
| API 응답 시간 | 평균 200ms 이하 | p95 레이턴시 |
| 동시 렌더링 | 최소 100개 병렬 처리 | Peak 시간대 처리량 |
| 가용성 | 99.5% 이상 | 월간 업타임 |

### 4.2 보안 요구사항
```
인증/인가:
├── OAuth 2.0 (Google, Kakao)
├── JWT 기반 세션 관리
├── 역할 기반 접근 제어 (RBAC)
└── API Key 관리

데이터 보호:
├── 전송 중 암호화 (TLS 1.3)
├── 저장 시 암호화 (AES-256)
├── 개인정보 마스킹
└── GDPR/PIPA 준수

인프라 보안:
├── WAF 적용
├── DDoS 방어
├── 침입 탐지 시스템
└── 정기 취약점 스캔
```

### 4.3 확장성 요구사항
```
수평 확장:
├── 컨테이너 오케스트레이션 (Kubernetes)
├── 서비스별 독립 스케일링
├── 지역별 CDN 배포
└── 데이터베이스 샤딩 지원

부하 분산:
├── Application Load Balancer
├── 큐 기반 작업 분산 (Redis Queue)
└── 캐시 전략 (Redis/CDN)
```

## 5. 기술 스택 명세

### 5.1 프론트엔드
```yaml
Framework: Next.js 14+ (App Router)
Language: TypeScript 5.x
Styling: Tailwind CSS + shadcn/ui
State: Zustand + TanStack Query
Forms: React Hook Form + Zod
Charts: Recharts
Animation: Framer Motion
```

### 5.2 백엔드
```yaml
Primary: Node.js (NestJS) / Python (FastAPI)
API: REST + GraphQL (선택적)
Auth: Passport.js + JWT
Validation: class-validator / Pydantic
Queue: BullMQ (Redis)
```

### 5.3 자동화/AI
```yaml
Orchestration: n8n (셀프호스팅)
LLM:
  - Google Gemini 3 Flash (대본 생성)
  - Gemini 2.5 Flash (요약/분석)
TTS:
  - ElevenLabs (프리미엄)
  - OpenAI TTS (표준)
Video Rendering:
  - Creatomate (템플릿 기반)
  - Shotstack (코드 기반)
Image:
  - Midjourney API
  - DALL-E 3
  - Stable Diffusion (셀프호스팅)
```

### 5.4 인프라
```yaml
Cloud: AWS / GCP
Container: Docker + Kubernetes (EKS/GKE)
Database:
  - PostgreSQL (Primary)
  - Redis (Cache/Queue)
  - MongoDB (비정형 데이터)
Storage:
  - S3/GCS (미디어 파일)
  - CloudFront/Cloud CDN
CI/CD: GitHub Actions
Monitoring:
  - Prometheus + Grafana
  - Sentry (Error Tracking)
  - DataDog (APM)
```
